# DESIGN_RULES.md

## WorknFlow Design Goal

WorknFlow should feel like a simple, trustworthy place to find practical AI workflows.

It should not feel like:

* a generic SaaS landing page
* enterprise documentation
* a developer-only tool
* a crypto dashboard
* an AI prompt dump
* a vibe-coded template

It should feel like:

* a clean student-friendly search tool
* a practical guide library
* a trustworthy workflow reference
* a product an average person can understand in 10 seconds

## Core UI Principle

Reduce explanation. Increase usefulness.

Every page should answer:

1. What can I do here?
2. Which workflow should I use?
3. Can I trust it?
4. What do I copy next?

## Voice and Copy Rules

Use plain language.

Avoid:

* workflow runbook
* inspectable
* judgment
* leverage
* unlock
* revolutionary
* supercharge
* seamless
* framework
* systematize
* optimize
* AI-powered magic

Prefer:

* workflow
* steps
* tested
* example
* check
* guide
* before you start
* what to copy
* what good output looks like
* use this when
* don’t use this when

Do not overuse uppercase eyebrow labels.

Use sentence case for most labels.

Bad:
VERIFIED WORKFLOW RUNBOOKS

Better:
Tested AI workflows

Bad:
A SMALL LIBRARY OF INSPECTABLE AI WORKFLOWS

Better:
Find a workflow that matches what you’re trying to do.

## Homepage Rules

The homepage should focus on search and usefulness, not product explanation.

The first screen should include:

* A plain headline
* A large search box
* A few suggested searches
* A short trust line
* 2 clear buttons maximum

Better headline:
Find the right AI workflow for what you’re trying to do.

Better subheadline:
Search tested workflows with example outputs, copy the steps, and check the result before trusting it.

Suggested searches:

* Make flashcards from messy notes
* Improve an essay argument
* Plan a small app idea
* Explain a wrong math solution

The homepage should not feel like a marketing page first. It should feel like a tool.

## Navigation Rules

Keep navigation minimal:

* Workflows
* Trust
* Request

Avoid making the header too tiny. It should feel intentional and usable.

## Workflow Library Rules

The library page should feel like a searchable catalog.

It should prioritize:

* search
* categories
* readable workflow cards
* clear trust signals

Cards should show:

* workflow title
* one-line use case
* category
* difficulty
* tested platform
* freshness
* 2–3 step preview
* clear “View workflow” action

Avoid too many badges. Too many badges make the page feel technical.

## Workflow Detail Page Rules

The workflow detail page should feel like a guide someone can actually follow.

Use this structure:

1. Title and short use case
2. Trust row: tested platform, freshness, difficulty, learning-safe
3. Before you start
4. Copy the steps
5. Example output
6. Check the result
7. Use this when / don’t use this when
8. Small verification details

Avoid making the page feel like developer documentation.

Prompt boxes should be readable, not huge code walls.

The copy button should be obvious.

The quality checklist should feel helpful, not like homework.

## Visual Rules

Use:

* off-white background
* white cards
* dark text
* muted gray supporting text
* one calm accent color
* fewer borders
* fewer tiny badges
* stronger spacing hierarchy
* larger useful elements
* fewer tiny labels

Avoid:

* too many boxes
* too much tiny text
* too many uppercase labels
* too many equal-weight cards
* pages that look like a wireframe
* giant empty white areas
* fake dashboards

## Average User Test

After every UI change, ask:

Would a normal student understand this page in 10 seconds?

If not, simplify.

Would this feel trustworthy to someone who is not technical?

If not, reduce jargon and make the trust signals clearer.

Would this feel useful before it feels impressive?

If not, make search and workflows more prominent.

## Phase 3 UI Correction Goal

The current UI is clean but too generic and too technical.

Fix it by:

* removing most eyebrow text
* making the homepage search-first
* simplifying copy
* reducing badge overload
* making workflow pages feel like step-by-step guides
* making trust signals clear but not overwhelming
* making the site feel usable by an average person, not only a builder
